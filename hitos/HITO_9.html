Contexto e integracion

El Hito 9 extiende los patrones estructurales del Hito 8 y agrega mecanismos para enriquecer envios, simplificar subsistemas, compartir memoria y controlar acceso a recursos costosos.

HitoPatronesAporte al sistema
Hito 8Adapter + Bridge + CompositeIntegracion externa, reportes desacoplados y jerarquia logistica.
Hito 9Decorator + Facade + Flyweight + ProxyServicios opcionales, interfaz unificada, ahorro de memoria y persistencia diferida con cache.
Decorator

Compone servicios como seguro, GPS, SMS y prioridad sin tocar el envio base.

Facade

Coordina inventario, pagos, rastreo, reportes y notificaciones desde una sola API.

Flyweight

Comparte ubicaciones repetidas para no instanciar millones de objetos iguales.

Proxy

Inicializa el repositorio real bajo demanda y cachea consultas frecuentes.

Actividad 1 · Decorator

Servicios opcionales en envios

Se modelo un contrato de envio desacoplado del dominio previo para evitar colision con `com.logismart.dominio.Envio` y para poder componer servicios sin alterar hitos anteriores.

GoF · Decorator
public interface Envio {
    double obtenerCosto();
    int obtenerTiempoEntrega();
    String obtenerDescripcion();
    String obtenerServicios();
}

public class EnvioBasico implements Envio {
    public double obtenerCosto() { return peso * 10.0; }
    public int obtenerTiempoEntrega() { return 3; }
}
public class DecoradorSeguro extends DecoradorEnvio {
    public double obtenerCosto() {
        return envioDecorado.obtenerCosto() * 1.15;
    }
}

public class DecoradorPrioritario extends DecoradorEnvio {
    public int obtenerTiempoEntrega() {
        return Math.max(1, envioDecorado.obtenerTiempoEntrega() - 2);
    }
}

Actividad 2 · Facade

Operacion compleja encapsulada

`ServicioLogisticaFacade` centraliza la creacion, cancelacion y consulta de estado de un envio. Internamente coordina 5 subsistemas, pero el cliente usa solo 3 metodos de alto nivel.

GoF · Facade
public class ServicioLogisticaFacade {
    public String crearEnvio(String productoId, double monto, String email, String telefono) {
        if (!inventario.verificarStock(productoId)) return null;
        if (!pagos.procesarPago("TARJETA", monto)) return null;
        inventario.restarStock(productoId, 1);
        String numero = rastreo.crearNumeroSeguimiento();
        notificaciones.enviarEmail(email, ...);
        notificaciones.enviarSMS(telefono, ...);
        reportes.generarReporte("ENVIO", numero, "PDF");
        return numero;
    }
}

Actividad 4 · Proxy

Repositorio con lazy loading y cache

`ProxyRepositorioEnvios` retrasa la conexion al repositorio real hasta la primera consulta y conserva resultados frecuentes en cache por id y cache de lista.

GoF · Proxy
public class ProxyRepositorioEnvios implements RepositorioEnvios {
    private RepositorioEnviosReal repositorioReal;
    private final Map<String, Envio> cache = new HashMap<>();
    private List<Envio> cacheEnvios;

    private RepositorioEnviosReal obtenerRepositorio() {
        if (repositorioReal == null) {
            repositorioReal = new RepositorioEnviosReal();
        }
        return repositorioReal;
    }
}

Conclusion

Hito 9 deja a LogiSmart con una capa estructural mas madura: servicios opcionales componibles, flujos complejos encapsulados, ubicaciones compartidas y persistencia diferida con cache. El sistema queda mejor preparado para futuras clases de comportamiento sin perder trazabilidad con lo hecho en hitos previos.
