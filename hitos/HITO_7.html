Abstract Factory — Implementación

Se implementó LogiSmartApp que recibe la región y delega toda la creación a la factory correspondiente.

GoF · Abstract Factory

2.1 Clase LogiSmartApp

public class LogiSmartApp {
    private final LogiSmartFactory factory;
    private final Vehiculo vehiculo;
    private final CalculadorCostos calculador;
    private final ProveedorMapas mapas;
    private final Logger logger;

    public LogiSmartApp(String region) {
        this.logger = Logger.getInstance();

        // Una sola decisión de región → toda la familia de objetos es consistente
        if (region.equalsIgnoreCase("Argentina"))
            this.factory = new LogiSmartFactoryArgentina();
        else if (region.equalsIgnoreCase("Brasil"))
            this.factory = new LogiSmartFactoryBrasil();
        else
            throw new IllegalArgumentException("Región no soportada: " + region);

        this.vehiculo   = factory.crearVehiculo();       // Auto o Moto
        this.calculador = factory.crearCalculadorCostos(); // IVA 21% o ICMS 12%
        this.mapas      = factory.crearProveedorMapas();   // Google Maps AR o HERE Maps

        logger.info("LogiSmartApp inicializado para: " + region);
    }

    public void procesarEnvio(String origen, String destino, double peso) {
        logger.info("Procesando envío de " + origen + " a " + destino);
        vehiculo.conducir();
        double costo = calculador.calcularCosto(peso);
        mapas.obtenerRuta(origen, destino);
        logger.info("Costo calculado: $" + String.format("%.2f", costo));
    }
}

2.2 Por qué usar Abstract Factory aquí

Sin Abstract FactoryCon Abstract Factory
El cliente crea cada objeto por separado y puede mezclar objetos de distintas regiones (Auto argentino + ICMS brasileño). La factory garantiza coherencia: si la región es Argentina, todos los objetos son de Argentina.
Agregar Chile requiere cambiar código del cliente. Agregar Chile = nueva clase LogiSmartFactoryChile. El cliente no cambia.
✅ Uso correcto de factory · ✅ Manejo de región inválida · ✅ Objetos coherentes entre sí

Prototype — Clonación eficiente

Envio implementa Cloneable. El método clone() permite crear 100 copias sin invocar el Builder cada vez.

GoF · Prototype

5.1 Implementación de clone()

public class Envio implements Cloneable {

    @Override
    public Envio clone() {
        try {
            Envio clonado = (Envio) super.clone(); // copia superficial
            // ordenes es una List mutable → se copia la referencia de la lista
            clonado.ordenes = new ArrayList<>(this.ordenes);
            return clonado;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException("Error al clonar Envio", e);
        }
    }

    // Setter necesario para modificar el id del clon (Prototype requiere identidad propia)
    public void setId(String id) { this.id = id; }
}
¿Por qué super.clone() es suficiente? Todos los atributos de Envio son primitivos (double, boolean) o inmutables (String, LocalTime), excepto ordenes. Esta lista se copia explícitamente para que el clon tenga su propio historial de órdenes.

5.2 Clonación masiva (100 envíos)

// 1. Construir el prototipo UNA SOLA VEZ con el Builder
Envio prototipo = new Envio.EnvioBuilder("ENV-PROTO", "Buenos Aires", "Córdoba")
        .descripcion("Medicinas")
        .peso(2.5)
        .fragil(true)
        .build();

// 2. Clonar 100 veces → O(1) por clon, no se invoca el Builder
List<Envio> envios = new ArrayList<>();
for (int i = 0; i < 100; i++) {
    Envio clon = prototipo.clone();
    clon.setId("ENV-" + (1000 + i)); // única diferencia entre clones
    envios.add(clon);
}
// prototipo.getId() sigue siendo "ENV-PROTO" — no se modificó

6.2 LogiSmartApp

public class LogiSmartApp {
    private final LogiSmartFactory factory;   // Abstract Factory
    private final Logger logger;              // Singleton
    private final ConfiguracionSistema config; // Singleton
    private final List<Envio> envios;
    private final List<Usuario> usuarios;

    public LogiSmartApp(String region) {
        this.logger  = Logger.getInstance();           // Singleton
        this.config  = ConfiguracionSistema.getInstance(); // Singleton
        this.factory = crearFactory(region);            // Abstract Factory
        this.envios  = new ArrayList<>();
        this.usuarios = new ArrayList<>();
    }

    // Builder — crea un Envio con atributos declarativos
    public void crearEnvio(String origen, String destino) {
        Envio e = new Envio.EnvioBuilder("ENV-" + envios.size(), origen, destino)
                .descripcion("Envío estándar").build();
        envios.add(e);
    }

    // Prototype — clona un prototipo N veces
    public void crearEnviosMultiples(int cantidad) {
        Envio proto = new Envio.EnvioBuilder("PROTO", "Buenos Aires", "Córdoba")
                .peso(1.0).build();
        for (int i = 0; i < cantidad; i++) {
            Envio clon = proto.clone();
            clon.setId("ENV-" + (1000 + i));
            envios.add(clon);
        }
    }

    // Factory Method — crea usuario usando clase existente del dominio
    public void crearUsuario(String tipo, String nombre) {
        Usuario u = UsuarioFactory.crearUsuario(tipo, nombre); // ClienteFinal, OperadorLogistico, AdminEmpresa
        usuarios.add(u);
        u.saludar(); // polimorfismo: cada subclase sabe cómo saludarse
    }

    // Abstract Factory — procesa con objetos coherentes para la región
    public void procesarEnvio(String origen, String destino, double peso) {
        Vehiculo v      = factory.crearVehiculo();
        CalculadorCostos c = factory.crearCalculadorCostos();
        ProveedorMapas m   = factory.crearProveedorMapas();
        v.conducir();
        double costo = c.calcularCosto(peso);
        m.obtenerRuta(origen, destino);
    }
}
