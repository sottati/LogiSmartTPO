🎯 Clase 11: Patrones de Comportamiento I

Chain of Responsibility, Command, Interpreter

Objetivo de la Clase

Estudiar patrones que definen cómo los objetos interactúan y distribuyen responsabilidades:

  • Pasar solicitudes a través de una cadena de manejadores
  • Encapsular solicitudes como objetos
  • Definir lenguajes específicos del dominio

¿Qué son Patrones de Comportamiento?

Los patrones de comportamiento se ocupan de la comunicación efectiva entre objetos y la asignación de responsabilidades. A diferencia de los patrones estructurales (que se ocupan de la composición de clases y objetos), los patrones de comportamiento caracterizan los flujos complejos de control que son difíciles de seguir en tiempo de ejecución.

Categorías de Patrones de Comportamiento

Categoría Patrones Enfoque
Responsabilidad Chain of Responsibility, Command, Interpreter Cómo distribuir responsabilidades entre objetos
Encapsulación Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor Cómo encapsular comportamiento

Pregunta Central

¿Cómo puedo pasar solicitudes a través de múltiples objetos? ¿Cómo encapsulo una solicitud como un objeto? ¿Cómo defino un lenguaje específico?

⛓️ Chain of Responsibility

Problema: Múltiples Manejadores

Un envío en LogiSmart necesita ser aprobado por múltiples niveles:

  • Validador de Datos (verifica que los datos sean correctos)
  • Validador de Inventario (verifica que hay stock)
  • Validador de Pago (verifica que el pago es válido)
  • Validador de Seguridad (verifica que no hay restricciones)

Problema: ¿Quién maneja cada validación? ¿En qué orden?

Cliente │ ├─→ ¿Datos válidos? │ ├─→ No → Error │ └─→ Sí → Siguiente │ ├─→ ¿Stock disponible? │ ├─→ No → Error │ └─→ Sí → Siguiente │ ├─→ ¿Pago válido? │ ├─→ No → Error │ └─→ Sí → Siguiente │ └─→ ¿Seguridad OK? ├─→ No → Error └─→ Sí → Aprobado

Solución: Chain of Responsibility

Crea una cadena de manejadores donde cada uno decide si procesa la solicitud o la pasa al siguiente.

┌──────────────────────────────────────────────────────────┐ │ Cliente │ └──────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────┐ │ ValidadorDatos │ │ - Valida datos │ │ - Si OK, pasa al siguiente │ │ - Si error, rechaza │ └──────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────┐ │ ValidadorInventario │ │ - Verifica stock │ │ - Si OK, pasa al siguiente │ │ - Si error, rechaza │ └──────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────┐ │ ValidadorPago │ │ - Verifica pago │ │ - Si OK, pasa al siguiente │ │ - Si error, rechaza │ └──────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────┐ │ ValidadorSeguridad │ │ - Verifica restricciones │ │ - Si OK, aprueba │ │ - Si error, rechaza │ └──────────────────────────────────────────────────────────┘

Implementación

Paso 1: Interfaz del Manejador

public abstract class ValidadorEnvio {
    protected ValidadorEnvio siguiente;
    
    public void setSiguiente(ValidadorEnvio siguiente) {
        this.siguiente = siguiente;
    }
    
    /**
     * Procesa la solicitud o la pasa al siguiente
     */
    public abstract boolean validar(Envio envio);
}

Paso 2: Manejadores Concretos

// Validador 1: Datos
public class ValidadorDatos extends ValidadorEnvio {
    @Override
    public boolean validar(Envio envio) {
        System.out.println("[ValidadorDatos] Validando datos...");
        
        if (envio.getOrigen() == null || envio.getDestino() == null) {
            System.err.println("✗ Datos inválidos");
            return false;
        }
        
        System.out.println("✓ Datos válidos");
        
        if (siguiente != null) {
            return siguiente.validar(envio);
        }
        return true;
    }
}

// Validador 2: Inventario
public class ValidadorInventario extends ValidadorEnvio {
    @Override
    public boolean validar(Envio envio) {
        System.out.println("[ValidadorInventario] Verificando stock...");
        
        if (envio.getPeso() > 100) { // Simulación
            System.err.println("✗ Stock insuficiente");
            return false;
        }
        
        System.out.println("✓ Stock disponible");
        
        if (siguiente != null) {
            return siguiente.validar(envio);
        }
        return true;
    }
}

// Validador 3: Pago
public class ValidadorPago extends ValidadorEnvio {
    @Override
    public boolean validar(Envio envio) {
        System.out.println("[ValidadorPago] Verificando pago...");
        
        if (envio.getCosto() <= 0) {
            System.err.println("✗ Pago inválido");
            return false;
        }
        
        System.out.println("✓ Pago válido");
        
        if (siguiente != null) {
            return siguiente.validar(envio);
        }
        return true;
    }
}

// Validador 4: Seguridad
public class ValidadorSeguridad extends ValidadorEnvio {
    @Override
    public boolean validar(Envio envio) {
        System.out.println("[ValidadorSeguridad] Verificando restricciones...");
        
        if (envio.getDestino().contains("Restringido")) {
            System.err.println("✗ Destino restringido");
            return false;
        }
        
        System.out.println("✓ Seguridad OK");
        
        if (siguiente != null) {
            return siguiente.validar(envio);
        }
        return true;
    }
}

Paso 3: Construcción de la Cadena

// Crear validadores
ValidadorEnvio validador1 = new ValidadorDatos();
ValidadorEnvio validador2 = new ValidadorInventario();
ValidadorEnvio validador3 = new ValidadorPago();
ValidadorEnvio validador4 = new ValidadorSeguridad();

// Construir cadena
validador1.setSiguiente(validador2);
validador2.setSiguiente(validador3);
validador3.setSiguiente(validador4);

// Usar la cadena
Envio envio = new Envio("Buenos Aires", "Córdoba", 5.0, 150.0);
if (validador1.validar(envio)) {
    System.out.println("✓ Envío aprobado");
} else {
    System.out.println("✗ Envío rechazado");
}

Ventajas

  • Desacoplamiento: Cliente no conoce los manejadores específicos
  • Flexibilidad: Fácil agregar, remover o reordenar manejadores
  • Responsabilidad Única: Cada manejador hace una cosa

Desventajas

  • Debugging Difícil: Difícil seguir el flujo
  • Sin Garantía: La solicitud podría no ser manejada
  • Rendimiento: Múltiples llamadas pueden ser lentas
Tiempo estimado: 25 minutos

📋 Command Pattern

Problema: Solicitudes como Parámetros

LogiSmart necesita encolar, deshacer y registrar operaciones:

  • Crear envío
  • Cancelar envío
  • Actualizar estado
  • Generar reporte

Problema: ¿Cómo encapsular estas operaciones para poder encolarlas, deshacerlas o registrarlas?

Solución: Command

Encapsula una solicitud como un objeto que puede ser parametrizado, encolado, registrado y deshecho.

┌──────────────────────────────────────────────────────────┐ │ Cliente │ │ - Crea comandos │ │ - Los encola │ └──────────────────────────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────────────────────────┐ │ Comando (Interfaz) │ │ - ejecutar() │ │ - deshacer() │ └──────────────────────────────────────────────────────────┘ │ ┌────┴────┬────────────┬──────────────┐ ▼ ▼ ▼ ▼ ┌─────────┐ ┌──────────┐ ┌────────────┐ ┌──────────────┐ │CrearEnv │ │CancelarEn│ │ActualizarEs│ │GenerarReport │ │ │ │ │ │ │ │ │ └─────────┘ └──────────┘ └────────────┘ └──────────────┘

Implementación

Paso 1: Interfaz Comando

public interface Comando {
    void ejecutar();
    void deshacer();
    String obtenerDescripcion();
}

Paso 2: Comandos Concretos

// Comando: Crear Envío
public class ComandoCrearEnvio implements Comando {
    private ServicioEnvios servicio;
    private Envio envio;
    private String numeroSeguimiento;
    
    public ComandoCrearEnvio(ServicioEnvios servicio, Envio envio) {
        this.servicio = servicio;
        this.envio = envio;
    }
    
    @Override
    public void ejecutar() {
        System.out.println("[Comando] Creando envío...");
        numeroSeguimiento = servicio.crearEnvio(envio);
        System.out.println("✓ Envío creado: " + numeroSeguimiento);
    }
    
    @Override
    public void deshacer() {
        System.out.println("[Comando] Deshaciendo: Cancelando envío...");
        servicio.cancelarEnvio(numeroSeguimiento);
        System.out.println("✓ Envío cancelado");
    }
    
    @Override
    public String obtenerDescripcion() {
        return "Crear envío de " + envio.getOrigen() + " a " + envio.getDestino();
    }
}

// Comando: Cancelar Envío
public class ComandoCancelarEnvio implements Comando {
    private ServicioEnvios servicio;
    private String numeroSeguimiento;
    
    public ComandoCancelarEnvio(ServicioEnvios servicio, String numeroSeguimiento) {
        this.servicio = servicio;
        this.numeroSeguimiento = numeroSeguimiento;
    }
    
    @Override
    public void ejecutar() {
        System.out.println("[Comando] Cancelando envío: " + numeroSeguimiento);
        servicio.cancelarEnvio(numeroSeguimiento);
        System.out.println("✓ Envío cancelado");
    }
    
    @Override
    public void deshacer() {
        System.out.println("[Comando] Deshaciendo: Reactivando envío...");
        servicio.reactivarEnvio(numeroSeguimiento);
        System.out.println("✓ Envío reactivado");
    }
    
    @Override
    public String obtenerDescripcion() {
        return "Cancelar envío: " + numeroSeguimiento;
    }
}

// Comando: Actualizar Estado
public class ComandoActualizarEstado implements Comando {
    private ServicioEnvios servicio;
    private String numeroSeguimiento;
    private String nuevoEstado;
    private String estadoAnterior;
    
    public ComandoActualizarEstado(ServicioEnvios servicio, String numeroSeguimiento, String nuevoEstado) {
        this.servicio = servicio;
        this.numeroSeguimiento = numeroSeguimiento;
        this.nuevoEstado = nuevoEstado;
    }
    
    @Override
    public void ejecutar() {
        System.out.println("[Comando] Actualizando estado a: " + nuevoEstado);
        estadoAnterior = servicio.obtenerEstado(numeroSeguimiento);
        servicio.actualizarEstado(numeroSeguimiento, nuevoEstado);
        System.out.println("✓ Estado actualizado");
    }
    
    @Override
    public void deshacer() {
        System.out.println("[Comando] Deshaciendo: Restaurando estado anterior...");
        servicio.actualizarEstado(numeroSeguimiento, estadoAnterior);
        System.out.println("✓ Estado restaurado: " + estadoAnterior);
    }
    
    @Override
    public String obtenerDescripcion() {
        return "Actualizar estado a: " + nuevoEstado;
    }
}

Paso 3: Invocador (Cola de Comandos)

public class ColaComandos {
    private List historial = new ArrayList<>();
    private int indiceActual = -1;
    
    /**
     * Ejecuta un comando y lo agrega al historial
     */
    public void ejecutar(Comando comando) {
        // Eliminar comandos deshacidos
        while (indiceActual < historial.size() - 1) {
            historial.remove(historial.size() - 1);
        }
        
        comando.ejecutar();
        historial.add(comando);
        indiceActual++;
    }
    
    /**
     * Deshace el último comando
     */
    public void deshacer() {
        if (indiceActual >= 0) {
            Comando comando = historial.get(indiceActual);
            comando.deshacer();
            indiceActual--;
            System.out.println("✓ Deshecho: " + comando.obtenerDescripcion());
        }
    }
    
    /**
     * Rehace el comando deshacido
     */
    public void rehacer() {
        if (indiceActual < historial.size() - 1) {
            indiceActual++;
            Comando comando = historial.get(indiceActual);
            comando.ejecutar();
            System.out.println("✓ Rehecho: " + comando.obtenerDescripcion());
        }
    }
    
    /**
     * Muestra el historial
     */
    public void mostrarHistorial() {
        System.out.println("\n=== Historial de Comandos ===");
        for (int i = 0; i < historial.size(); i++) {
            String marca = i == indiceActual ? "→" : " ";
            System.out.println(marca + " " + (i + 1) + ". " + historial.get(i).obtenerDescripcion());
        }
    }
}

Paso 4: Uso del Command

ColaComandos cola = new ColaComandos();
ServicioEnvios servicio = new ServicioEnvios();

// Crear envío
Envio envio = new Envio("Buenos Aires", "Córdoba", 5.0, 150.0);
Comando cmd1 = new ComandoCrearEnvio(servicio, envio);
cola.ejecutar(cmd1);

// Actualizar estado
Comando cmd2 = new ComandoActualizarEstado(servicio, "ENV-001", "EN TRÁNSITO");
cola.ejecutar(cmd2);

// Actualizar estado nuevamente
Comando cmd3 = new ComandoActualizarEstado(servicio, "ENV-001", "ENTREGADO");
cola.ejecutar(cmd3);

cola.mostrarHistorial();

// Deshacer
cola.deshacer(); // Vuelve a "EN TRÁNSITO"
cola.deshacer(); // Vuelve a "CONFIRMADO"

// Rehacer
cola.rehacer(); // Vuelve a "EN TRÁNSITO"

Ventajas

  • Desacoplamiento: Separar objeto que invoca de quien ejecuta
  • Undo/Redo: Fácil implementar deshacer/rehacer
  • Cola de Operaciones: Encolar, registrar, ejecutar después
  • Macros: Combinar comandos en secuencias

Desventajas

  • Complejidad: Muchas clases pequeñas
  • Memoria: Historial puede consumir mucha memoria
  • Overhead: Pequeña sobrecarga por encapsulación
Tiempo estimado: 25 minutos

🔤 Interpreter Pattern

Problema: Lenguajes Específicos del Dominio

LogiSmart necesita un lenguaje para definir reglas de envío:

ORIGEN = "Buenos Aires" AND DESTINO = "Córdoba" AND PESO < 10 ORIGEN = "Rosario" OR DESTINO = "Mendoza" COSTO > 100 AND PRIORIDAD = "ALTA"

Problema: ¿Cómo interpretar y evaluar estas expresiones?

Solución: Interpreter

Define una representación gramatical para el lenguaje y un intérprete para procesarlo.

Expresión: "ORIGEN = 'Buenos Aires' AND PESO < 10" │ ▼ ┌───────────────────┐ │ Parser │ │ (Análisis) │ └───────────────────┘ │ ▼ ┌───────────────────┐ │ Árbol de Sintaxis│ │ │ │ AND │ │ / \ │ │ = < │ │ / \ / \ │ │O B P 10 │ └───────────────────┘ │ ▼ ┌───────────────────┐ │ Intérprete │ │ (Evaluación) │ └───────────────────┘ │ ▼ Resultado: true/false

Implementación

Paso 1: Interfaz Expresión

public interface Expresion {
    boolean evaluar(Envio envio);
}

// Contexto (datos a evaluar)
public class ContextoEnvio {
    private Envio envio;
    
    public ContextoEnvio(Envio envio) {
        this.envio = envio;
    }
    
    public Envio obtenerEnvio() {
        return envio;
    }
}

Paso 2: Expresiones Terminales

// Expresión: ORIGEN = valor
public class ExpresionOrigen implements Expresion {
    private String valor;
    
    public ExpresionOrigen(String valor) {
        this.valor = valor;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        return envio.getOrigen().equals(valor);
    }
}

// Expresión: DESTINO = valor
public class ExpresionDestino implements Expresion {
    private String valor;
    
    public ExpresionDestino(String valor) {
        this.valor = valor;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        return envio.getDestino().equals(valor);
    }
}

// Expresión: PESO < valor
public class ExpresionPeso implements Expresion {
    private double valor;
    private String operador; // "<", ">", "="
    
    public ExpresionPeso(double valor, String operador) {
        this.valor = valor;
        this.operador = operador;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        switch (operador) {
            case "<": return envio.getPeso() < valor;
            case ">": return envio.getPeso() > valor;
            case "=": return envio.getPeso() == valor;
            default: return false;
        }
    }
}

Paso 3: Expresiones No-Terminales (Operadores)

// Expresión: AND
public class ExpresionAND implements Expresion {
    private Expresion izquierda;
    private Expresion derecha;
    
    public ExpresionAND(Expresion izquierda, Expresion derecha) {
        this.izquierda = izquierda;
        this.derecha = derecha;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        return izquierda.evaluar(envio) && derecha.evaluar(envio);
    }
}

// Expresión: OR
public class ExpresionOR implements Expresion {
    private Expresion izquierda;
    private Expresion derecha;
    
    public ExpresionOR(Expresion izquierda, Expresion derecha) {
        this.izquierda = izquierda;
        this.derecha = derecha;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        return izquierda.evaluar(envio) || derecha.evaluar(envio);
    }
}

// Expresión: NOT
public class ExpresionNOT implements Expresion {
    private Expresion expresion;
    
    public ExpresionNOT(Expresion expresion) {
        this.expresion = expresion;
    }
    
    @Override
    public boolean evaluar(Envio envio) {
        return !expresion.evaluar(envio);
    }
}

Paso 4: Uso del Interpreter

// Crear expresiones
Expresion origen = new ExpresionOrigen("Buenos Aires");
Expresion peso = new ExpresionPeso(10, "<");
Expresion expresion1 = new ExpresionAND(origen, peso);

// Evaluar
Envio envio1 = new Envio("Buenos Aires", "Córdoba", 5.0, 150.0);
Envio envio2 = new Envio("Rosario", "Córdoba", 15.0, 200.0);

System.out.println("Envío 1 cumple regla: " + expresion1.evaluar(envio1)); // true
System.out.println("Envío 2 cumple regla: " + expresion1.evaluar(envio2)); // false

// Expresión más compleja
Expresion destino = new ExpresionDestino("Córdoba");
Expresion expresion2 = new ExpresionOR(
    new ExpresionAND(origen, peso),
    destino
);

System.out.println("Envío 1 cumple expresión 2: " + expresion2.evaluar(envio1)); // true
System.out.println("Envío 2 cumple expresión 2: " + expresion2.evaluar(envio2)); // true

Ventajas

  • Flexibilidad: Fácil agregar nuevas reglas
  • Separación: Lógica de negocio separada de interpretación
  • Reutilización: Las expresiones pueden reutilizarse

Desventajas

  • Complejidad: Muchas clases para gramáticas complejas
  • Rendimiento: Interpretación puede ser lenta
  • Mantenimiento: Cambios en la gramática requieren cambios en código
Tiempo estimado: 20 minutos

¿Qué es un Diagrama de Secuencia?

Un diagrama de secuencia muestra cómo los objetos interactúan en una secuencia temporal. Es especialmente útil para visualizar patrones de comportamiento.

Tabla Comparativa

Conceptos Clave

Chain of Responsibility

  • Cadena de manejadores que procesan solicitudes
