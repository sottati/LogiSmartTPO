🔄 Iterator Pattern

Problema: Acceso a Colecciones

LogiSmart tiene diferentes estructuras de datos para almacenar envíos:

  • Array de envíos
  • Lista enlazada de envíos
  • Árbol de envíos por región
  • Tabla hash de envíos por ID

Problema: ¿Cómo iterar sobre ellas sin conocer su estructura interna?

Solución: Iterator

Proporciona una forma de acceder secuencialmente a los elementos de una colección sin exponer su estructura.

┌─────────────────────────────────────┐ │ Cliente │ └─────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ Iterator (Interfaz) │ │ - siguiente() │ │ - tieneSiguiente() │ │ - obtenerActual() │ └─────────────────────────────────────┘ │ ┌────┴────┬──────────┬────────────┐ ▼ ▼ ▼ ▼ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │Array │ │Lista │ │Árbol │ │Hash │ │Iterator│ │Iterator│ │Iterator│ │Iterator│ └────────┘ └────────┘ └────────┘ └────────┘

Implementación

Paso 1: Interfaz Iterator

public interface IteradorEnvios { boolean tieneSiguiente(); Envio obtenerSiguiente(); void reiniciar(); }

Paso 2: Interfaz Colección

public interface ColeccionEnvios { IteradorEnvios crearIterador(); void agregar(Envio envio); void remover(Envio envio); }

Paso 3: Implementaciones Concretas

// Colección: Array public class ColeccionArray implements ColeccionEnvios { private Envio[] envios = new Envio[100]; private int tamaño = 0; @Override public IteradorEnvios crearIterador() { return new IteradorArray(); } @Override public void agregar(Envio envio) { envios[tamaño++] = envio; } private class IteradorArray implements IteradorEnvios { private int indice = 0; @Override public boolean tieneSiguiente() { return indice < tamaño; } @Override public Envio obtenerSiguiente() { return envios[indice++]; } @Override public void reiniciar() { indice = 0; } } } // Colección: Lista Enlazada public class ColeccionLista implements ColeccionEnvios { private Nodo cabeza; private int tamaño = 0; @Override public IteradorEnvios crearIterador() { return new IteradorLista(); } @Override public void agregar(Envio envio) { Nodo nuevoNodo = new Nodo(envio); if (cabeza == null) { cabeza = nuevoNodo; } else { Nodo actual = cabeza; while (actual.siguiente != null) { actual = actual.siguiente; } actual.siguiente = nuevoNodo; } tamaño++; } private class IteradorLista implements IteradorEnvios { private Nodo actual = cabeza; @Override public boolean tieneSiguiente() { return actual != null; } @Override public Envio obtenerSiguiente() { Envio envio = actual.envio; actual = actual.siguiente; return envio; } @Override public void reiniciar() { actual = cabeza; } } private class Nodo { Envio envio; Nodo siguiente; Nodo(Envio envio) { this.envio = envio; } } }

Paso 4: Uso del Iterator

// Usar colección sin conocer su estructura ColeccionEnvios coleccion = new ColeccionArray(); coleccion.agregar(new Envio("Buenos Aires", "Córdoba", 5.0, 150.0, "TARJETA", "PROD-001")); coleccion.agregar(new Envio("Rosario", "Mendoza", 8.0, 200.0, "EFECTIVO", "PROD-002")); // Iterar IteradorEnvios iterador = coleccion.crearIterador(); while (iterador.tieneSiguiente()) { Envio envio = iterador.obtenerSiguiente(); System.out.println(envio.getOrigen() + " → " + envio.getDestino()); }

Ventajas

  • Encapsulación: Estructura interna oculta
  • Flexibilidad: Cambiar estructura sin afectar cliente
  • Múltiples Iteradores: Varios iteradores simultáneos

Desventajas

  • Complejidad: Más clases
  • Rendimiento: Pequeña sobrecarga
Tiempo estimado: 20 minutos

🤝 Mediator Pattern

Problema: Comunicación Compleja

En LogiSmart, múltiples componentes necesitan comunicarse:

  • Centro de distribución comunica con validador
  • Validador comunica con sistema de pago
  • Sistema de pago comunica con notificador
  • Notificador comunica con centro

Problema: Cada componente conoce a todos los demás → acoplamiento alto

Sin Mediator (Acoplamiento Alto): Centro ←→ Validador ↕ ↕ Pago ←→ Notificador Con Mediator (Desacoplamiento): Centro ↓ Mediator ← Validador ↑ Pago → Notificador

Solución: Mediator

Centraliza la comunicación entre objetos en un mediador.

Implementación

Paso 1: Interfaz Mediator

public interface MediadorEnvios { void registrarCentro(CentroDistribucion centro); void registrarValidador(ValidadorEnvio validador); void registrarPago(SistemaPago pago); void registrarNotificador(SistemaNotificacion notificador); void notificar(String evento, Object datos); }

Paso 2: Mediator Concreto

public class MediadorEnviosConcreto implements MediadorEnvios { private CentroDistribucion centro; private ValidadorEnvio validador; private SistemaPago pago; private SistemaNotificacion notificador; @Override public void registrarCentro(CentroDistribucion centro) { this.centro = centro; } @Override public void registrarValidador(ValidadorEnvio validador) { this.validador = validador; } @Override public void registrarPago(SistemaPago pago) { this.pago = pago; } @Override public void registrarNotificador(SistemaNotificacion notificador) { this.notificador = notificador; } @Override public void notificar(String evento, Object datos) { switch (evento) { case "ENVIO_CREADO": Envio envio = (Envio) datos; System.out.println("[Mediator] Envío creado: " + envio.getId()); validador.validar(envio); break; case "VALIDACION_OK": System.out.println("[Mediator] Validación OK"); pago.procesarPago((Envio) datos); break; case "PAGO_CONFIRMADO": System.out.println("[Mediator] Pago confirmado"); notificador.enviarConfirmacion((Envio) datos); break; case "NOTIFICACION_ENVIADA": System.out.println("[Mediator] Notificación enviada"); centro.registrarEnvio((Envio) datos); break; } } }

Paso 3: Componentes que Usan Mediator

// Centro de Distribución public class CentroDistribucion { private MediadorEnvios mediador; public CentroDistribucion(MediadorEnvios mediador) { this.mediador = mediador; } public void crearEnvio(Envio envio) { System.out.println("[Centro] Creando envío..."); mediador.notificar("ENVIO_CREADO", envio); } public void registrarEnvio(Envio envio) { System.out.println("[Centro] Registrando envío en sistema"); } } // Validador public class ValidadorEnvio { private MediadorEnvios mediador; public ValidadorEnvio(MediadorEnvios mediador) { this.mediador = mediador; } public void validar(Envio envio) { System.out.println("[Validador] Validando envío..."); if (envio.getCosto() > 0) { System.out.println("[Validador] ✓ Válido"); mediador.notificar("VALIDACION_OK", envio); } } } // Sistema de Pago public class SistemaPago { private MediadorEnvios mediador; public SistemaPago(MediadorEnvios mediador) { this.mediador = mediador; } public void procesarPago(Envio envio) { System.out.println("[Pago] Procesando pago..."); System.out.println("[Pago] ✓ Pago confirmado"); mediador.notificar("PAGO_CONFIRMADO", envio); } } // Sistema de Notificación public class SistemaNotificacion { private MediadorEnvios mediador; public SistemaNotificacion(MediadorEnvios mediador) { this.mediador = mediador; } public void enviarConfirmacion(Envio envio) { System.out.println("[Notificador] Enviando confirmación..."); System.out.println("[Notificador] ✓ Confirmación enviada"); mediador.notificar("NOTIFICACION_ENVIADA", envio); } }

Paso 4: Uso del Mediator

// Crear mediador MediadorEnvios mediador = new MediadorEnviosConcreto(); // Crear componentes CentroDistribucion centro = new CentroDistribucion(mediador); ValidadorEnvio validador = new ValidadorEnvio(mediador); SistemaPago pago = new SistemaPago(mediador); SistemaNotificacion notificador = new SistemaNotificacion(mediador); // Registrar componentes mediador.registrarCentro(centro); mediador.registrarValidador(validador); mediador.registrarPago(pago); mediador.registrarNotificador(notificador); // Usar Envio envio = new Envio("Buenos Aires", "Córdoba", 5.0, 150.0, "TARJETA", "PROD-001"); centro.crearEnvio(envio);

Ventajas

  • Desacoplamiento: Componentes no se conocen entre sí
  • Centralización: Lógica de comunicación en un lugar
  • Reutilización: Componentes reutilizables

Desventajas

  • Complejidad: Mediator puede volverse complejo
  • Punto Único de Fallo: Si mediator falla, todo falla
Tiempo estimado: 20 minutos

💾 Memento Pattern

Problema: Capturar Estado

LogiSmart necesita guardar el estado de un envío en diferentes momentos:

  • Estado inicial (CONFIRMADO)
  • Estado en tránsito (EN_TRANSITO)
  • Estado entregado (ENTREGADO)

Problema: ¿Cómo guardar y restaurar estados sin violar encapsulación?

Solución: Memento

Captura el estado interno de un objeto sin violar su encapsulación.

┌──────────────────────────────────────┐ │ Envio (Originador) │ │ - estado │ │ - crearMemento() │ │ - restaurarDesdeMemento() │ └──────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────┐ │ Memento (Snapshot) │ │ - estado (privado) │ │ - obtenerEstado() │ └──────────────────────────────────────┘ │ ▼ ┌──────────────────────────────────────┐ │ Cuidador (Historial) │ │ - mementos[] │ │ - guardar() │ │ - restaurar() │ └──────────────────────────────────────┘

Implementación

Paso 1: Memento

public class MementoEnvio { private String estado; private String origen; private String destino; private double peso; private double costo; private long timestamp; public MementoEnvio(String estado, String origen, String destino, double peso, double costo) { this.estado = estado; this.origen = origen; this.destino = destino; this.peso = peso; this.costo = costo; this.timestamp = System.currentTimeMillis(); } public String obtenerEstado() { return estado; } public String obtenerOrigen() { return origen; } public String obtenerDestino() { return destino; } public double obtenerPeso() { return peso; } public double obtenerCosto() { return costo; } public long obtenerTimestamp() { return timestamp; } }

Paso 2: Originador

public class Envio { private String id; private String estado; private String origen; private String destino; private double peso; private double costo; public Envio(String id, String origen, String destino, double peso, double costo) { this.id = id; this.origen = origen; this.destino = destino; this.peso = peso; this.costo = costo; this.estado = "CONFIRMADO"; } /** * Crea un memento del estado actual */ public MementoEnvio crearMemento() { return new MementoEnvio(estado, origen, destino, peso, costo); } /** * Restaura el estado desde un memento */ public void restaurarDesdeMemento(MementoEnvio memento) { this.estado = memento.obtenerEstado(); this.origen = memento.obtenerOrigen(); this.destino = memento.obtenerDestino(); this.peso = memento.obtenerPeso(); this.costo = memento.obtenerCosto(); } public void cambiarEstado(String nuevoEstado) { this.estado = nuevoEstado; } public String obtenerEstado() { return estado; } public String getId() { return id; } }

Paso 3: Cuidador

public class HistorialEnvios { private List historial = new ArrayList<>(); private int indiceActual = -1; /** * Guarda el estado actual */ public void guardarEstado(Envio envio) { // Eliminar estados posteriores while (indiceActual < historial.size() - 1) { historial.remove(historial.size() - 1); } historial.add(envio.crearMemento()); indiceActual++; System.out.println("✓ Estado guardado: " + envio.obtenerEstado()); } /** * Restaura a un estado anterior */ public void irAlEstadoAnterior(Envio envio) { if (indiceActual > 0) { indiceActual--; envio.restaurarDesdeMemento(historial.get(indiceActual)); System.out.println("✓ Restaurado a: " + envio.obtenerEstado()); } } /** * Restaura a un estado posterior */ public void irAlEstadoSiguiente(Envio envio) { if (indiceActual < historial.size() - 1) { indiceActual++; envio.restaurarDesdeMemento(historial.get(indiceActual)); System.out.println("✓ Restaurado a: " + envio.obtenerEstado()); } } /** * Muestra el historial */ public void mostrarHistorial() { System.out.println("\n=== Historial de Estados ==="); for (int i = 0; i < historial.size(); i++) { MementoEnvio m = historial.get(i); String marca = i == indiceActual ? "→" : " "; System.out.println(marca + " " + (i + 1) + ". " + m.obtenerEstado()); } } }

Paso 4: Uso del Memento

Envio envio = new Envio("ENV-001", "Buenos Aires", "Córdoba", 5.0, 150.0); HistorialEnvios historial = new HistorialEnvios(); // Guardar estado inicial historial.guardarEstado(envio); // Cambiar estado envio.cambiarEstado("EN_TRANSITO"); historial.guardarEstado(envio); // Cambiar estado nuevamente envio.cambiarEstado("ENTREGADO"); historial.guardarEstado(envio); historial.mostrarHistorial(); // Ir al estado anterior historial.irAlEstadoAnterior(envio); historial.mostrarHistorial();

Ventajas

  • Encapsulación: Estado interno protegido
  • Historial: Fácil navegar por estados
  • Undo/Redo: Implementación natural

Desventajas

  • Memoria: Puede consumir mucha memoria
  • Complejidad: Más clases
Tiempo estimado: 15 minutos

👁️ Observer Pattern

Problema: Notificación de Cambios

LogiSmart necesita notificar a múltiples objetos cuando cambia el estado de un envío:

  • Centro de distribución
  • Sistema de notificación
  • Sistema de auditoría
  • Dashboard en tiempo real

Problema: ¿Cómo notificar sin acoplamiento?

Solución: Observer

Define una relación uno-a-muchos donde cuando un objeto cambia, todos sus observadores son notificados automáticamente.

┌──────────────────────────────────────┐ │ Sujeto (Envio) │ │ - observadores[] │ │ - adjuntarObservador() │ │ - desadjuntarObservador() │ │ - notificarObservadores() │ └──────────────────────────────────────┘ │ ┌────┴────┬──────────┬────────────┐ ▼ ▼ ▼ ▼ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │Centro │ │Notif │ │Auditoría│ │Dashboard│ │ │ │ │ │ │ │ │ └────────┘ └────────┘ └────────┘ └────────┘

Implementación

Paso 1: Interfaz Observador

public interface ObservadorEnvio { void actualizar(Envio envio); }

Paso 2: Sujeto (Observable)

public class Envio { private String id; private String estado; private List observadores = new ArrayList<>(); public Envio(String id, String estado) { this.id = id; this.estado = estado; } /** * Adjunta un observador */ public void adjuntarObservador(ObservadorEnvio observador) { observadores.add(observador); System.out.println("✓ Observador adjuntado"); } /** * Desadjunta un observador */ public void desadjuntarObservador(ObservadorEnvio observador) { observadores.remove(observador); System.out.println("✓ Observador desadjuntado"); } /** * Notifica a todos los observadores */ private void notificarObservadores() { for (ObservadorEnvio observador : observadores) { observador.actualizar(this); } } /** * Cambia el estado y notifica */ public void cambiarEstado(String nuevoEstado) { this.estado = nuevoEstado; System.out.println("[Envio] Estado cambiado a: " + nuevoEstado); notificarObservadores(); } public String obtenerEstado() { return estado; } public String getId() { return id; } }

Paso 3: Observadores Concretos

// Observador: Centro de Distribución public class CentroDistribucionObservador implements ObservadorEnvio { @Override public void actualizar(Envio envio) { System.out.println("[Centro] Envío " + envio.getId() + " cambió a: " + envio.obtenerEstado()); } } // Observador: Sistema de Notificación public class SistemaNotificacionObservador implements ObservadorEnvio { @Override public void actualizar(Envio envio) { System.out.println("[Notificador] Enviando email sobre cambio de estado..."); System.out.println("[Notificador] Email enviado para envío: " + envio.getId()); } } // Observador: Sistema de Auditoría public class SistemaAuditoriaObservador implements ObservadorEnvio { @Override public void actualizar(Envio envio) { System.out.println("[Auditoría] Registrando cambio de estado en log..."); System.out.println("[Auditoría] Envío: " + envio.getId() + " | Estado: " + envio.obtenerEstado()); } } // Observador: Dashboard en Tiempo Real public class DashboardObservador implements ObservadorEnvio { @Override public void actualizar(Envio envio) { System.out.println("[Dashboard] Actualizando interfaz en tiempo real..."); System.out.println("[Dashboard] Envío: " + envio.getId() + " | Estado: " + envio.obtenerEstado()); } }

Paso 4: Uso del Observer

// Crear envío Envio envio = new Envio("ENV-001", "CONFIRMADO"); // Crear observadores ObservadorEnvio centro = new CentroDistribucionObservador(); ObservadorEnvio notificador = new SistemaNotificacionObservador(); ObservadorEnvio auditoria = new SistemaAuditoriaObservador(); ObservadorEnvio dashboard = new DashboardObservador(); // Adjuntar observadores envio.adjuntarObservador(centro); envio.adjuntarObservador(notificador); envio.adjuntarObservador(auditoria); envio.adjuntarObservador(dashboard); // Cambiar estado (notifica a todos) envio.cambiarEstado("EN_TRANSITO"); System.out.println(); envio.cambiarEstado("ENTREGADO");

Ventajas

  • Desacoplamiento: Sujeto no conoce observadores específicos
  • Notificación Automática: Cambios se propagan automáticamente
  • Escalabilidad: Fácil agregar nuevos observadores

Desventajas

  • Orden Impredecible: Orden de notificación no garantizado
  • Memoria: Observadores pueden acumular referencias
  • Debugging Difícil: Flujo de control no obvio
Tiempo estimado: 15 minutos

⚡ Arquitecturas Event-Driven

¿Qué es Event-Driven?

Una arquitectura event-driven es un patrón arquitectónico donde los componentes se comunican a través de eventos.

Componentes Principales

  • Productor de Eventos: Genera eventos
  • Evento: Representa algo que sucedió
  • Consumidor de Eventos: Reacciona a eventos
  • Bus de Eventos: Distribuye eventos

Implementación con Observer

// Evento public class EventoEnvio { private String tipo; private Envio envio; private long timestamp; public EventoEnvio(String tipo, Envio envio) { this.tipo = tipo; this.envio = envio; this.timestamp = System.currentTimeMillis(); } public String obtenerTipo() { return tipo; } public Envio obtenerEnvio() { return envio; } public long obtenerTimestamp() { return timestamp; } } // Bus de Eventos public class BusEventos { private List consumidores = new ArrayList<>(); public void registrarConsumidor(ConsumidorEventos consumidor) { consumidores.add(consumidor); } public void publicarEvento(EventoEnvio evento) { System.out.println("[Bus] Publicando evento: " + evento.obtenerTipo()); for (ConsumidorEventos consumidor : consumidores) { consumidor.procesarEvento(evento); } } } // Consumidor de Eventos public interface ConsumidorEventos { void procesarEvento(EventoEnvio evento); } // Consumidor Concreto public class ConsumidorNotificacion implements ConsumidorEventos { @Override public void procesarEvento(EventoEnvio evento) { if ("ENVIO_ENTREGADO".equals(evento.obtenerTipo())) { System.out.println("[Notificación] Enviando confirmación de entrega..."); } } }

Ventajas de Event-Driven

  • Desacoplamiento Total: Productores y consumidores no se conocen
  • Escalabilidad: Fácil agregar nuevos consumidores
  • Asincronía: Procesamiento no bloqueante
  • Auditabilidad: Todos los eventos se registran

Casos de Uso

  • Sistemas de notificación en tiempo real
  • Procesamiento de transacciones
  • Integración de sistemas
  • IoT y sensores
Tiempo estimado: 10 minutos

📈 Comparación de Patrones

Tabla Comparativa

Patrón Propósito Cuándo Usar
Iterator Acceder a elementos sin exponer estructura Múltiples colecciones con diferentes estructuras
Mediator Centralizar comunicación entre objetos Muchos objetos que necesitan comunicarse
Memento Capturar y restaurar estado Necesitas undo/redo o historial
Observer Notificar múltiples objetos de cambios Un objeto cambia y otros necesitan reaccionar

Matriz de Decisión

  • ¿Múltiples colecciones? → Iterator
  • ¿Comunicación compleja? → Mediator
  • ¿Necesitas historial? → Memento
  • ¿Cambios que notificar? → Observer
Tiempo estimado: 5 minutos
