Clase 3: Del Diseño a la Implementación

Traduciendo Diagramas de Clases UML a Código Java en Eclipse

🔄Del Diseño a la Implementación

En la clase anterior, nos convertimos en arquitectos de software. Diseñamos la estructura de LogiSmart usando Diagramas de Clases UML. Hoy, cambiamos de sombrero. Nos convertimos en constructores. Vamos a tomar esos planos que diseñaron y empezar a construir la estructura de nuestro edificio de software.

¿Por Qué Separamos Diseño de Implementación?

En la ingeniería de software profesional, el diseño es una actividad deliberada y crítica. Al diseñar primero en UML, logramos varias cosas importantes:

🧠 Pensamiento Abstracto

Nos enfocamos en la estructura sin distracciones de sintaxis. Es como pensar en la arquitectura de un edificio sin preocuparse por qué tipo de cemento usar.

🌐 Comunicación Universal

Un diagrama UML es entendido por equipos en diferentes lenguajes. Java, Python, C# - todos pueden implementar el mismo diseño.

🐛 Detección Temprana de Errores

Es infinitamente más barato corregir una flecha en un diagrama que refactorizar cientos de líneas de código mal estructurado.

🦅 Visión Global

El diagrama nos da una vista de pájaro de todo el sistema, algo muy difícil de ver cuando estás inmerso en el código.

🛠️Eclipse IDE: Nuestra Herramienta de Construcción

Así como un carpintero tiene su taller, nosotros, los desarrolladores de software, tenemos nuestro IDE (Entorno de Desarrollo Integrado). Un IDE es mucho más que un simple editor de texto. Es una suite completa de herramientas que nos ayuda a escribir, compilar, depurar y gestionar nuestro código de manera eficiente.

¿Por Qué Eclipse?

Existen muchos IDEs en el mercado: IntelliJ IDEA, Visual Studio Code, NetBeans. Nosotros usaremos Eclipse, uno de los IDEs más robustos y utilizados en el mundo empresarial para el desarrollo en Java. Es de código abierto, gratuito y tiene una comunidad enorme.

¿Qué nos ofrece Eclipse?

📝 Editor Inteligente

Autocompletado, resaltado de sintaxis, detección de errores en tiempo real. El editor sabe Java y nos ayuda a escribir mejor código.

⚙️ Compilador Integrado

Traduce nuestro código Java a bytecode que la máquina virtual de Java (JVM) puede ejecutar. Todo automáticamente.

🐛 Depurador (Debugger)

Una herramienta increíblemente poderosa para ejecutar código línea por línea, inspeccionar variables y encontrar errores.

📦 Gestión de Proyectos

Organiza archivos, librerías y dependencias en una estructura clara. Todo en un lugar.

Instalación de Eclipse

Descargar desde eclipse.org. Seleccionar "Eclipse IDE for Java Developers". Seguir el instalador. ¡Listo!

📁Creando un Proyecto Java en Eclipse

Lo primero es crear nuestro espacio de trabajo. En Eclipse, esto se llama un Proyecto. Un proyecto es como una carpeta especial que contiene todo nuestro código, librerías y configuración.

Paso a Paso: Crear un Proyecto

  1. File > New > Java Project - Abre el diálogo de creación
  2. Project Name: Le damos un nombre, por ejemplo, LogiSmartTPO
  3. JRE: Nos aseguramos de que esté seleccionada una versión de Java (JavaSE-11 o superior)
  4. Project Layout: Dejamos la opción por defecto. Esto crea dos carpetas: src (código fuente) y bin (código compilado)
  5. Finish - ¡Proyecto creado!

Creando Paquetes (Packages)

Para mantener el orden, crearemos paquetes. Un paquete es como una carpeta para nuestras clases. Es una convención usar un nombre de dominio invertido. Por ejemplo: com.logismart.dominio

  1. Click derecho en la carpeta src en el Package Explorer
  2. New > Package
  3. Name: com.logismart.dominio
  4. Finish

Estructura de Proyecto Recomendada

Para un proyecto más grande, podrías tener múltiples paquetes:

  • com.logismart.dominio - Clases del modelo de dominio
  • com.logismart.servicio - Servicios y lógica de negocio
  • com.logismart.persistencia - Acceso a base de datos
  • com.logismart.ui - Interfaz de usuario

📦Traduciendo Clases UML a Código Java

Ahora viene la magia. Tomemos una clase de nuestro diagrama y la traducimos a código. La traducción es casi 1 a 1.

Creando una Clase en Eclipse

  1. Click derecho en el paquete com.logismart.dominio
  2. New > Class
  3. Name: Vehiculo
  4. Finish

Eclipse nos crea el esqueleto:

package com.logismart.dominio; public class Vehiculo { }

De UML a Java: La Traducción Directa

Veamos cómo se traduce un diagrama UML a código Java. Usaremos la clase Vehiculo como ejemplo.

📋 En UML (Diagrama)

Clase: Vehiculo

Atributos:

  • - id: String
  • - patente: String
  • - marca: String
  • - capacidadPeso: double

💻 En Java (Código)

Clase: public class Vehiculo

Atributos:

  • private String id;
  • private String patente;
  • private String marca;
  • private double capacidadPeso;

Como ven, visibilidad nombre: tipo en UML se convierte en visibilidad tipo nombre; en Java. El - se convierte en private. El + se convertirá en public.

📝Atributos y Constructores

Ahora agregamos los atributos a nuestra clase. Todos deben ser privados (encapsulación).

public class Vehiculo { // Atributos (privados - encapsulación) private String id; private String patente; private String marca; private String modelo; private int anio; private double capacidadPeso; private double capacidadVolumen; }

¿Qué es un Constructor?

Un constructor es un método especial que se llama cuando creamos un nuevo objeto (new Vehiculo()). Sirve para inicializar el estado del objeto. Si no definimos uno, Java nos da uno por defecto vacío. Pero es una buena práctica definir el nuestro.

public class Vehiculo { private String id; private String patente; private String marca; private double capacidadPeso; // Constructor public Vehiculo(String id, String patente, String marca, double capacidadPeso) { this.id = id; this.patente = patente; this.marca = marca; this.capacidadPeso = capacidadPeso; } }

La palabra clave this se usa para desambiguar entre el parámetro del método (patente) y el atributo de la clase (this.patente).

Usando el Constructor

Ahora podemos crear objetos Vehiculo:

Vehiculo v1 = new Vehiculo("V001", "ABC-123", "Toyota", 1000.0); Vehiculo v2 = new Vehiculo("V002", "XYZ-789", "Ford", 1500.0);

🔑Getters y Setters: Accediendo a Atributos Privados

¿Recuerdan la encapsulación? Los atributos son privados. ¿Cómo accedemos a ellos desde fuera? A través de métodos públicos llamados getters (para obtener el valor) y setters (para modificar el valor).

¿Qué es un Getter?

Un getter es un método que retorna el valor de un atributo privado. La convención es nombrarlos get + nombre del atributo.

public class Vehiculo { private String patente; // Getter para la patente public String getPatente() { return this.patente; } }

¿Qué es un Setter?

Un setter es un método que modifica el valor de un atributo privado. La convención es nombrarlos set + nombre del atributo.

public class Vehiculo { private String marca; // Setter para la marca public void setMarca(String nuevaMarca) { this.marca = nuevaMarca; } }

Generando Getters y Setters Automáticamente

Escribir getters y setters es tedioso. Eclipse puede hacerlo por nosotros automáticamente:

  1. Click derecho en el código de la clase
  2. Source > Generate Getters and Setters...
  3. Seleccionar los atributos para los que quieres generar getters/setters
  4. OK - ¡Eclipse genera todo el código!

Resultado Automático

Eclipse genera algo como esto:

public String getId() { return id; } public void setId(String id) { this.id = id; } public String getPatente() { return patente; } public void setPatente(String patente) { this.patente = patente; }

Usando Getters y Setters

Ahora podemos acceder y modificar atributos de forma controlada:

Vehiculo v1 = new Vehiculo("V001", "ABC-123", "Toyota", 1000.0); // Usar getter String patente = v1.getPatente(); // "ABC-123" // Usar setter v1.setMarca("Honda"); // Ahora la marca cambió System.out.println(v1.getMarca()); // "Honda"

🔗Traduciendo Relaciones: Asociación y Composición

Ahora, lo interesante: las relaciones. ¿Cómo representamos que una clase "conoce" o "tiene" a otra?

Relación de Asociación Simple

Recordemos la relación entre Ruta y Vehiculo. Una Ruta tiene un Vehiculo. En el diagrama, es una línea de asociación. En código, esto se traduce en un atributo cuyo tipo es la otra clase.

📋 En UML

Ruta -------- Vehiculo (1 a 1)

💻 En Java

public class Ruta { private Vehiculo vehiculoAsignado; }

Relación de Composición (1 a Muchos)

Si una Ruta se compone de muchos PuntosDeEntrega (relación 1 a 0..*), usamos una Colección, como una List.

import java.util.ArrayList; import java.util.List; public class Ruta { private String id; private Vehiculo vehiculoAsignado; // Composición con multiplicidad múltiple private List puntosDeEntrega; public Ruta(String id, Vehiculo vehiculo) { this.id = id; this.vehiculoAsignado = vehiculo; // Inicializamos la lista vacía this.puntosDeEntrega = new ArrayList<>(); } public void agregarPuntoDeEntrega(PuntoDeEntrega punto) { this.puntosDeEntrega.add(punto); } public List getPuntosDeEntrega() { return this.puntosDeEntrega; } }

¿Qué es ArrayList?

ArrayList es una clase de Java que representa una lista dinámica (puede crecer o encogerse). Es como un array, pero más flexible. List<PuntoDeEntrega> significa "una lista de objetos de tipo PuntoDeEntrega".

👨‍👧Herencia en Java: La Relación "Es un"

La herencia es otra relación fundamental. En UML, es una flecha con punta triangular. En Java, usamos la palabra clave extends.

Creando una Jerarquía de Usuarios

Recordemos nuestra jerarquía: Usuario es la clase base, Cliente y Operador heredan de ella.

Paso 1: Crear la Clase Base (Usuario)

public class Usuario { private String id; private String email; private String password; public Usuario(String id, String email, String password) { this.id = id; this.email = email; this.password = password; } public boolean autenticar(String passwordIngresado) { return this.password.equals(passwordIngresado); } // Getters y setters... }

Paso 2: Crear la Clase Hija (Cliente)

Para indicar que Cliente hereda de Usuario, usamos extends:

public class Cliente extends Usuario { // No necesito declarar id, email, password // ¡Ya los heredé de Usuario! private String razonSocial; private String cuit; public Cliente(String id, String email, String password, String razonSocial) { // Debo llamar al constructor de la clase padre (Usuario) super(id, email, password); this.razonSocial = razonSocial; } public String getRazonSocial() { return this.razonSocial; } }

La palabra clave super es la forma de acceder a miembros (constructores, métodos) de la clase padre. super(id, email, password) llama al constructor de Usuario.

Usando la Herencia

// Crear un cliente Cliente c1 = new Cliente("U001", "juan@logismart.com", "pass123", "Juan's Logistics"); // Usar métodos heredados de Usuario boolean autenticado = c1.autenticar("pass123"); // true // Usar métodos propios de Cliente String razon = c1.getRazonSocial(); // "Juan's Logistics"

La Prueba de Liskov

Un Cliente siempre puede ser tratado como un Usuario (porque hereda de él). Esto es la esencia de la herencia correcta.

⚙️Implementando Comportamiento: Métodos con Lógica

Hasta ahora, nuestras clases son solo contenedores de datos. Pero la POO se trata de juntar datos y comportamiento. Ahora debemos implementar la lógica dentro de los métodos.

Ejemplo: Método con Lógica de Negocio

Tomemos el método agregarPuntoDeEntrega() en la clase Ruta. Podría tener lógica de negocio:

public class Ruta { private Vehiculo vehiculoAsignado; private List puntosDeEntrega; public boolean agregarPuntoDeEntrega(PuntoDeEntrega nuevoPunto) { // Lógica de negocio: verificar capacidad del vehículo double pesoTotal = 0; // Sumar el peso de todos los puntos actuales for (PuntoDeEntrega p : this.puntosDeEntrega) { pesoTotal += p.getEnvio().getPeso(); } // Verificar si el nuevo punto cabe if (this.vehiculoAsignado.verificarCapacidad( pesoTotal + nuevoPunto.getEnvio().getPeso())) { this.puntosDeEntrega.add(nuevoPunto); System.out.println("Punto agregado exitosamente."); return true; } else { System.out.println("Error: Capacidad del vehículo excedida."); return false; } } }

Aquí vemos el poder de la POO. El objeto Ruta colabora con el objeto Vehiculo y el objeto PuntoDeEntrega para cumplir una tarea. Cada objeto es responsable de su propia lógica. Ruta no sabe cómo el vehículo verifica su capacidad, solo le pregunta. Esto es bajo acoplamiento.

🎯Hito 3: Implementación en Eclipse

Hemos visto cómo pasar de un plano a una estructura de código. Ahora es su turno. En el Hito 3, tomarán el Diagrama de Clases que diseñaron en el Hito 2 e lo implementarán en Java usando Eclipse.

¿Qué Deben Entregar?

Un proyecto de Eclipse funcional con:

  • La estructura de paquetes correcta
  • Todas las clases de su diagrama implementadas como archivos .java
  • Atributos con la visibilidad correcta (¡encapsulación!)
  • Constructores para inicializar sus objetos
  • Getters y Setters para todos los atributos
  • Relaciones implementadas: atributos para asociación/composición y extends para herencia
  • Métodos principales definidos (el esqueleto, no necesitan implementar toda la lógica compleja aún)

Objetivo del Hito 3

Esta es una tarea de traducción y atención al detalle. El objetivo es que al final del hito, tengan una base de código sólida y bien estructurada que refleje fielmente el diseño que crearon. Esta base será sobre la que construiremos en las próximas clases, aplicando patrones de diseño y lógica de negocio más compleja.
